  1. ¿Que es un sistema de control de versiones?

    Es una herramienta que sirve para guardar y controlar los cambios que se hacen en un proyecto (por ejemplo código). Permite regresar a versiones anteriores, ver qué se modificó y trabajar en equipo sin que se pierda información.

  2. ¿Cual es la diferencia entre git y github?

    Git es el programa que se instala en la computadora y que permite llevar el control de versiones de manera local. GitHub es una plataforma en línea donde se pueden subir los repositorios de Git para compartirlos y trabajar en equipo.

  3. ¿Cómo creo una rama en git?

    Si solo quieres crearla, hay que usar (git branch nombre-rama) pero si quieres crearla y al mismo tiempo entrar en ella (git checkout -b nombre-rama).

  4. ¿Cómo clono un repositorio?

    git clone https://github.com/usuario/repositorio.git

  5. ¿Cómo puedo bajar los ultimos cambios de un repositorio?

    git pull

  6. ¿Cómo puedo gener un tag?

    git tag nombre-tag y con el -m puedes añadir un mensaje.

  7. Cómo hago una mezcla de ramas?

    Primero me cambio a la rama donde quiero unir los cambios (git checkout main) después hago un merge (git merge nombre-rama).

  8. ¿Cómo puedo moverme a un commit especifico?

    git checkout id-del-commit

  9. ¿Cómo puedo borrar una rama?

    git branch -d nombre-rama

  10. ¿Cómo puedo borrar un tag?

    git tag -d nombre-tag

  11. ¿Cómo puedo ver los cambios que estan por enviarse al repositorio?

    git status

  12. ¿Para que sirve el comando git remote -v?

    Sirve para ver las direcciones (URL) de los repositorios remotos que tengo configurados.
